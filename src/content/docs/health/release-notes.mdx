### 1.1.0 - June 14, 2026
<RN type="feature">Menstruation/period tracking — new `DataType.MenstruationFlow` for reading, writing, and observing menstrual flow on Apple HealthKit and Android Health Connect</RN>
<RN type="feature">New categorical `MenstrualFlow` enum (`Unspecified`, `None`, `Light`, `Medium`, `Heavy`) and `MenstruationFlowResult` record</RN>
<RN type="feature">New `IHealthService.GetMenstruationFlow(start, end)` returns individual flow records (event-based — no `Interval` bucketing, unlike the numeric metrics)</RN>
<RN type="feature">New `IHealthService.Write(MenstruationFlowResult)` overload for logging daily flow</RN>
<RN type="feature" platform="iOS">Uses `HKCategoryType.MenstrualFlow`; the `IsCycleStart` flag is persisted via the `HKMetadataKeyMenstrualCycleStart` metadata, and the `None` flow level is supported</RN>
<RN type="feature" platform="Android">Uses Health Connect `MenstruationFlowRecord` with the `READ_MENSTRUATION` / `WRITE_MENSTRUATION` permissions; `None` maps to `Unspecified` and `IsCycleStart` is ignored (Health Connect models the period span as a separate record)</RN>

### 1.0.0 - May 2026
<RN type="feature">Initial release of `Shiny.Health` - a unified cross-platform health data API for .NET MAUI</RN>
<RN type="feature">Real-time health data observation via `IHealthService.Observe(DataType)` returning `IAsyncEnumerable<HealthResult>` — stream new samples as they are recorded</RN>
<RN type="feature" platform="iOS">Push-based observation using `HKAnchoredObjectQuery` with automatic update handler for immediate sample delivery</RN>
<RN type="feature" platform="Android">Polling-based observation using Health Connect change tokens with configurable polling interval (default 5 seconds)</RN>
<RN type="feature">Forward-only observation — only yields samples added after `Observe` is called, not historical data</RN>
<RN type="feature">Supports all 12 data types including blood pressure (`BloodPressureResult`) and sleep duration</RN>
<RN type="feature">Write support for all 12 health metrics via `IHealthService.Write` — write `NumericHealthResult` or `BloodPressureResult` to Apple HealthKit and Android Health Connect</RN>
<RN type="feature">New `PermissionType` flags enum (`Read`, `Write`, `ReadWrite`) for granular permission requests</RN>
<RN type="feature">New `RequestPermissions(PermissionType, params DataType[])` overload for requesting read, write, or both permissions</RN>
<RN type="feature" platform="iOS">Write support uses `HKQuantitySample`, `HKCategorySample` (sleep), and `HKCorrelation` (blood pressure)</RN>
<RN type="feature" platform="Android">Write support uses Health Connect `InsertRecords` with proper record types for each metric</RN>
<RN type="feature">12 health metrics supported: Step Count, Heart Rate, Calories, Distance, Weight, Height, Body Fat %, Resting Heart Rate, Blood Pressure, Oxygen Saturation, Sleep Duration, and Hydration</RN>
<RN type="feature">Time-bucketed aggregate queries with configurable `Interval` (Minutes, Hours, Days)</RN>
<RN type="feature">Unified permission management via `IHealthService.RequestPermissions`</RN>
<RN type="feature">Dedicated `BloodPressureResult` with separate Systolic and Diastolic values</RN>
<RN type="feature" platform="iOS">Apple HealthKit integration using statistics collections and category samples</RN>
<RN type="feature" platform="Android">Google Health Connect integration with aggregate queries and record reading</RN>
<RN type="feature">AOT-compatible implementation with no .NET reflection</RN>
<RN type="feature">Full cancellation token support on all async query methods</RN>
