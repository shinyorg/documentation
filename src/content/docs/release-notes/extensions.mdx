## Extensions

### 2.0 - TBD
<RN type="feature" breaking={true}>We now use module initializers and a global service registry along with a new defined method `services.AddShinyServiceRegistry()` that pulls in all registered values</RN>

### 1.3.1 - September 19, 2025
<RN type="fix">Type and AsSelf was being ignored if a base class is being used</RN>

### v1.3.0 - September 16, 2025
<RN type="enhancement">DI registration now supports `AsSelf` which allows an implementation to be registered directly</RN>

### 1.2.2 - July 26, 2025
<RN type="fix">Booleans with false were not binding properly on stores</RN>
<RN type="fix" platform="iOS">Secure storage would not correctly store or clear all values</RN>

### 1.2.1 - July 15 2025
<RN type="enhancement">Logging has been optionally enabled throughout the DI binding</RN>

### 1.2 - July 14, 2025
<RN type="enhancement">ServiceAttribute now has a TryAdd property that will call the appropriate service lifetime method</RN>
<RN type="enhancement">Additional SingletonAttribute, ScopedAttribute, and TransientAttribute to remove the need for multiple namespaces</RN>
<RN type="enhancement">Type property on service attribute allows you to override which interface is used in multiple interface scenarios</RN>

### 1.1 - July 13, 2025
<RN type="feature">ServiceAttribute now accepts a `Category`.  Categories allow you to pass an argument to the AddGeneratedServices that only get instantiate if you pass that category name.  Great for unit testing or dev vs production</RN>
<RN type="enhancement">Now works with records</RN>
<RN type="fix" breaking={true}>ServiceAttribute KeyedName name parameter was moved from the constructor to a property</RN>
