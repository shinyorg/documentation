title: Acknowledgements & Sound
description: Configure audio feedback and text-to-speech response modes

The `IAiConversationService` provides four acknowledgement modes that control how the AI delivers responses â€” from silent text-only to full text-to-speech.

## Acknowledgement Modes

```csharp
aiService.Acknowledgement = AiAcknowledgement.Full;
```

| Mode | Behavior |
|---|---|
| `None` | No audio feedback. Responses are text-only via the `AiResponded` event. |
| `AudioBlip` | Short sound effects play at state transitions (thinking, responding, ok, cancel, error). No text-to-speech. |
| `LessWordy` | Text-to-speech reads the response aloud. A system prompt is added requesting concise responses. |
| `Full` | Text-to-speech reads the full, unmodified response aloud. |

## Sound Effects

When `Acknowledgement` is set to `AudioBlip`, the service plays sounds at each state transition. Configure sounds with string file names and a resolver callback:

```csharp
// Set the resolver that converts file names to streams
aiService.SoundResolver = name => FileSystem.OpenAppPackageFileAsync(name);

// Assign sound file names
aiService.OkSound = "ok.mp3";
aiService.ThinkSound = "think.mp3";
aiService.RespondingSound = "responding.mp3";
aiService.CancelSound = "cancel.mp3";
aiService.ErrorSound = "error.mp3";
```

### Sound Trigger Points

| Sound | When it plays |
|---|---|
| `ThinkSound` | AI begins processing (state â†’ Thinking) |
| `RespondingSound` | AI starts streaming response (state â†’ Responding) |
| `OkSound` | Request completes successfully |
| `CancelSound` | Operation is cancelled |
| `ErrorSound` | An error occurs during processing |

### SoundResolver

The `SoundResolver` property is a `Func<string, Task<Stream>>?` that the service calls with the sound file name to get a playable audio stream. This keeps the library platform-agnostic â€” you provide the resolver appropriate for your platform:

**MAUI:**
```csharp
aiService.SoundResolver = name => FileSystem.OpenAppPackageFileAsync(name);
```

**Blazor / ASP.NET:**
```csharp
var env = app.Services.GetRequiredService<IWebHostEnvironment>();
aiService.SoundResolver = name =>
    Task.FromResult<Stream>(File.OpenRead(Path.Combine(env.WebRootPath, "sounds", name)));
```

:::note
Sound effects only play when `Acknowledgement` is `AudioBlip` **and** both `SoundResolver` is set and the corresponding sound property is non-null.
:::

## State Tracking

The service exposes its current state and fires events you can observe:

```csharp
// Current state
var state = aiService.Status; // Idle, Listening, Thinking, Responding

// State change event (receives the new AiState)
aiService.StatusChanged += state =>
{
    Console.WriteLine($"State: {state}");
};

// AI response event (fired when the AI completes its response)
aiService.AiResponded += response =>
{
    if (response.Response.Text is { } text)
        Console.WriteLine($"AI: {text}");

    Console.WriteLine($"Was read aloud: {response.WasReadAloud}");
};
```

The `AiResponse` record is fired once when the AI completes its response. The `WasReadAloud` flag lets your UI decide whether to display a visual notification for responses that were already spoken.
