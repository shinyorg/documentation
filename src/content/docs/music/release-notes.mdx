## v3

### 3.0 - TBD
<RN type="feature">`IMusicIdentifier` interface — identify songs by listening to audio through the device microphone</RN>
<RN type="feature">`MusicIdentificationResult` record — contains Title, Artist, Album, Genre, ArtworkUrl, MusicUrl, and ISRC for identified songs</RN>
<RN type="feature" platform="iOS">iOS song identification via ShazamKit (`SHSession` + `AVAudioEngine`), requires `com.apple.developer.shazamkit` entitlement</RN>
<RN type="feature">`PlayCount` property on `MusicMetadata` — number of times a track has been played</RN>
<RN type="feature">Playlist CRUD on `IMediaLibrary` — `CreatePlaylistAsync`, `RemovePlaylistAsync`, `AddTrackToPlaylistAsync`, `RemoveTrackFromPlaylistAsync`</RN>
<RN type="feature" platform="iOS">`IsExplicit` property now populated from `MPMediaItem.IsExplicitItem`</RN>
<RN type="feature" platform="iOS">`ContentUri` now populated from `MPMediaItem.AssetURL` — provides `ipod-library://` URL for non-DRM tracks</RN>
<RN type="feature" platform="iOS">`StoreId` populated with `MPMediaItem.PersistentID` — enables `IsPlayable` for all library tracks</RN>
<RN type="feature" platform="iOS">Mac Catalyst support via `net10.0-maccatalyst26.2` target framework</RN>
<RN type="feature" platform="iOS">Streaming subscription check via MusicKit `MusicSubscription.GetCurrentAsync`</RN>
<RN type="feature" platform="Android">Android genre metadata — tracks now include genre from `MediaStore.Audio.Genres`</RN>
<RN type="feature" platform="Android">Android play count tracking — automatically incremented on playback, stored in local JSON</RN>
<RN type="feature" platform="Android">Android custom playlist management — locally-stored JSON playlists merged with MediaStore playlists</RN>
<RN type="breaking">Removed `IMusicManager` interface and `Shiny.Music.Sqlite` package — playlist CRUD moved to `IMediaLibrary`, play counts moved to `MusicMetadata.PlayCount`</RN>
<RN type="breaking">Removed `AddMusicManagementSqlite()` extension method — no longer needed</RN>
<RN type="breaking">Removed `Volume` property from `IMusicPlayer` — `MPMusicPlayerController` does not support per-app volume control on Apple platforms</RN>
<RN type="breaking">Minimum iOS/Mac Catalyst version raised to 17.0</RN>
<RN type="breaking">Removed macOS and tvOS targets — only iOS, Mac Catalyst, and Android are supported</RN>
<RN type="breaking" platform="iOS">iOS playlist CRUD now uses locally-stored JSON playlists (same as Android) instead of MusicKit</RN>

## v2

### 2.0 - April 22, 2026
<RN type="feature">`ILyricsProvider` interface — fetch lyrics for any music track via dependency injection</RN>
<RN type="feature">`LyricsResult` record — contains `PlainLyrics` (plain text) and `SyncedLyrics` (synchronized LRC format with timestamps)</RN>
<RN type="feature">`LrcLibLyricsProvider` — default cross-platform lyrics provider using the free [LRCLIB](https://lrclib.net) service, with AOT-compatible source-generated JSON serialization</RN>
<RN type="feature">`GetAlbumArtPathAsync(trackId)` on `IMediaLibrary` — retrieve album artwork as a file path for use as an image source</RN>
<RN type="feature">`Volume` property on `IMusicPlayer` — get/set playback volume from 0.0 (silent) to 1.0 (full), with automatic clamping</RN>
<RN type="feature" platform="Android">Android lyrics via LRCLIB — matches tracks by artist, title, and duration</RN>
<RN type="feature" platform="Android">Android album art via MediaStore content URI</RN>
<RN type="feature" platform="Android">Android volume control via `MediaPlayer.SetVolume`</RN>
<RN type="feature" platform="iOS">iOS lyrics via LRCLIB</RN>
<RN type="feature" platform="iOS">iOS album art retrieval</RN>
<RN type="feature" platform="iOS">iOS volume control</RN>

## v1

### 1.3 - March 13, 2026
<RN type="feature">`GetPlaylistsAsync()` on `IMediaLibrary` — returns all playlists from the device music library with song counts, sorted alphabetically</RN>
<RN type="feature">`GetPlaylistTracksAsync(playlistId)` on `IMediaLibrary` — returns all tracks in a playlist, in playlist order</RN>
<RN type="feature">`PlaylistInfo` record — contains playlist Id, Name, and SongCount</RN>
<RN type="feature" platform="Android">Android playlist querying via `MediaStore.Audio.Playlists` and `MediaStore.Audio.Playlists.Members`</RN>
<RN type="feature" platform="iOS">iOS playlist querying via `MPMediaQuery.PlaylistsQuery` and `MPMediaPlaylist`</RN>

### 1.2 - March 13, 2026
<RN type="feature">`MusicFilter` class — combine Genre, Year, Decade, and SearchQuery criteria with AND logic to filter tracks</RN>
<RN type="feature">`GetTracksAsync(MusicFilter)` on `IMediaLibrary` — query tracks matching any combination of filter criteria</RN>
<RN type="feature">`GetYearsAsync()` on `IMediaLibrary` — returns distinct release years with track counts, with optional `MusicFilter` parameter</RN>
<RN type="feature">`GetDecadesAsync()` on `IMediaLibrary` — returns distinct decades (e.g., 1990s) with track counts, with optional `MusicFilter` parameter</RN>
<RN type="feature">`GroupedCount<T>` record — generic type pairing a grouped value with its track count, used by all grouping methods</RN>
<RN type="feature">`Year` property on `MusicMetadata` — the release year of the track (nullable)</RN>
<RN type="feature">`GetGenresAsync` now accepts an optional `MusicFilter` and returns `GroupedCount<string>` with track counts</RN>
<RN type="feature">Cross-dimensional browsing — get genres by decade, decades by genre, years by genre, etc.</RN>
<RN type="feature" platform="Android">Year metadata read from `MediaStore.Audio.Media.YEAR`</RN>
<RN type="feature" platform="iOS">Year metadata extracted from `MPMediaItem.ReleaseDate`</RN>

### 1.1 - March 12, 2026
<RN type="feature">`GetGenresAsync()` on `IMediaLibrary` — returns all distinct genre names from the user's music library, sorted alphabetically</RN>
<RN type="feature" platform="Android">Android genre querying via `MediaStore.Audio.Genres`</RN>
<RN type="feature" platform="iOS">iOS genre querying via `MPMediaQuery.GenresQuery`</RN>

### 1.0 - March 6, 2026
<RN type="feature">Permission management — request and check music library access using platform-native APIs</RN>
<RN type="feature">Music metadata querying — retrieve all tracks or search by title, artist, or album</RN>
<RN type="feature">Audio playback — play, pause, resume, and stop music files with state change and completion events</RN>
<RN type="feature">File copying — copy music files to app-local storage (where DRM permits)</RN>
<RN type="feature">Dependency injection — `AddShinyMusic()` extension method for `IServiceCollection` registration</RN>
<RN type="feature">`HasStreamingSubscriptionAsync()` — check for active Apple Music subscription capability on iOS</RN>
<RN type="feature">`StoreId` property on `MusicMetadata` — Apple Music catalog ID enabling streaming playback of subscription content</RN>
<RN type="feature" platform="Android">Android implementation using `MediaStore.Audio.Media` and `Android.Media.MediaPlayer`</RN>
<RN type="feature" platform="iOS">iOS implementation using `MPMediaQuery` and `AVAudioPlayer`</RN>
<RN type="feature" platform="iOS">Streaming playback of Apple Music subscription tracks via `MPMusicPlayerController.SystemMusicPlayer` using `StoreId`</RN>
<RN type="feature" platform="iOS">DRM-aware — `ContentUri` is empty for Apple Music subscription tracks that cannot be copied, but streaming playback is available via `StoreId`</RN>
<RN type="feature" platform="iOS">`IsExplicit` property on `MusicMetadata` — reports whether a track is marked as explicit content via `MPMediaItem.IsExplicitItem`</RN>
