## v3

### 3.0 - May 3, 2026
<RN type="feature">Mac Catalyst support ‚Äî all features now work on macOS via Mac Catalyst in addition to iOS and Android</RN>
<RN type="feature">Removed `Microsoft.Maui.Essentials` dependency ‚Äî the library no longer requires .NET MAUI, enabling use in plain .NET for iOS/Android projects</RN>
<RN type="feature">Custom playlist CRUD on `IMediaLibrary` ‚Äî `CreatePlaylistAsync`, `RemovePlaylistAsync`, `AddTrackToPlaylistAsync`, `RemoveTrackFromPlaylistAsync` for app-managed playlists stored locally as JSON</RN>
<RN type="feature">`PlayCount` property on `MusicMetadata` ‚Äî number of times a track has been played</RN>
<RN type="feature" platform="iOS">Play counts from `MPMediaItem.PlayCount` on Apple platforms</RN>
<RN type="feature" platform="Android">Play counts tracked locally and auto-incremented on `PlayAsync` on Android</RN>
<RN type="feature" platform="Android">Android runtime permission handling via `ActivityProvider` ‚Äî no longer requires MAUI Permissions API</RN>
<RN type="feature" platform="iOS">Streaming subscription check now uses MusicKit `MusicSubscription` instead of `SKCloudServiceController`</RN>
<RN type="breaking">Removed `Volume` property from `IMusicPlayer`</RN>
<RN type="breaking">Removed `IMusicIdentifier` and ShazamKit song identification</RN>

## v2

### 2.0 - April 22, 2026
<RN type="feature">`ILyricsProvider` interface ‚Äî fetch lyrics for any music track via dependency injection</RN>
<RN type="feature">`LyricsResult` record ‚Äî contains `PlainLyrics` (plain text) and `SyncedLyrics` (synchronized LRC format with timestamps)</RN>
<RN type="feature">`LrcLibLyricsProvider` ‚Äî default cross-platform lyrics provider using the free [LRCLIB](https://lrclib.net) service, with AOT-compatible source-generated JSON serialization</RN>
<RN type="feature">`GetAlbumArtPathAsync(trackId)` on `IMediaLibrary` ‚Äî retrieve album artwork as a file path for use as an image source</RN>
<RN type="feature">`Volume` property on `IMusicPlayer` ‚Äî get/set playback volume from 0.0 (silent) to 1.0 (full), with automatic clamping</RN>
<RN type="feature" platform="Android">Android lyrics via LRCLIB ‚Äî matches tracks by artist, title, and duration</RN>
<RN type="feature" platform="Android">Android album art via MediaStore content URI</RN>
<RN type="feature" platform="Android">Android volume control via `MediaPlayer.SetVolume`</RN>
<RN type="feature" platform="iOS">iOS lyrics via LRCLIB (default) and `MPMediaItem.Lyrics` (native, via `IMediaLibrary` cast)</RN>
<RN type="feature" platform="iOS">iOS album art via `MPMediaItem.Artwork` ‚Äî exported as cached JPEG (600x600, 85% quality)</RN>
<RN type="feature" platform="iOS">iOS volume control via `AVPlayer.Volume`</RN>

## v1

### 1.3 - March 13, 2026
<RN type="feature">`GetPlaylistsAsync()` on `IMediaLibrary` ‚Äî returns all playlists from the device music library with song counts, sorted alphabetically</RN>
<RN type="feature">`GetPlaylistTracksAsync(playlistId)` on `IMediaLibrary` ‚Äî returns all tracks in a playlist, in playlist order</RN>
<RN type="feature">`PlaylistInfo` record ‚Äî contains playlist Id, Name, and SongCount</RN>
<RN type="feature" platform="Android">Android playlist querying via `MediaStore.Audio.Playlists` and `MediaStore.Audio.Playlists.Members`</RN>
<RN type="feature" platform="iOS">iOS playlist querying via `MPMediaQuery.PlaylistsQuery` and `MPMediaPlaylist`</RN>

### 1.2 - March 13, 2026
<RN type="feature">`MusicFilter` class ‚Äî combine Genre, Year, Decade, and SearchQuery criteria with AND logic to filter tracks</RN>
<RN type="feature">`GetTracksAsync(MusicFilter)` on `IMediaLibrary` ‚Äî query tracks matching any combination of filter criteria</RN>
<RN type="feature">`GetYearsAsync()` on `IMediaLibrary` ‚Äî returns distinct release years with track counts, with optional `MusicFilter` parameter</RN>
<RN type="feature">`GetDecadesAsync()` on `IMediaLibrary` ‚Äî returns distinct decades (e.g., 1990s) with track counts, with optional `MusicFilter` parameter</RN>
<RN type="feature">`GroupedCount<T>` record ‚Äî generic type pairing a grouped value with its track count, used by all grouping methods</RN>
<RN type="feature">`Year` property on `MusicMetadata` ‚Äî the release year of the track (nullable)</RN>
<RN type="feature">`GetGenresAsync` now accepts an optional `MusicFilter` and returns `GroupedCount<string>` with track counts</RN>
<RN type="feature">Cross-dimensional browsing ‚Äî get genres by decade, decades by genre, years by genre, etc.</RN>
<RN type="feature" platform="Android">Year metadata read from `MediaStore.Audio.Media.YEAR`</RN>
<RN type="feature" platform="iOS">Year metadata extracted from `MPMediaItem.ReleaseDate`</RN>

### 1.1 - March 12, 2026
<RN type="feature">`GetGenresAsync()` on `IMediaLibrary` ‚Äî returns all distinct genre names from the user's music library, sorted alphabetically</RN>
<RN type="feature" platform="Android">Android genre querying via `MediaStore.Audio.Genres`</RN>
<RN type="feature" platform="iOS">iOS genre querying via `MPMediaQuery.GenresQuery`</RN>

### 1.0 - March 6, 2026
<RN type="feature">Permission management ‚Äî request and check music library access using platform-native APIs</RN>
<RN type="feature">Music metadata querying ‚Äî retrieve all tracks or search by title, artist, or album</RN>
<RN type="feature">Audio playback ‚Äî play, pause, resume, and stop music files with state change and completion events</RN>
<RN type="feature">File copying ‚Äî copy music files to app-local storage (where DRM permits)</RN>
<RN type="feature">Dependency injection ‚Äî `AddShinyMusic()` extension method for `IServiceCollection` registration</RN>

<RN type="feature" platform="Android">Android implementation using `MediaStore.Audio.Media` and `Android.Media.MediaPlayer`</RN>
<RN type="feature" platform="iOS">iOS implementation using `MPMediaQuery` and `AVAudioPlayer`</RN>
<RN type="feature" platform="iOS">Streaming playback of Apple Music subscription tracks via `MPMusicPlayerController.SystemMusicPlayer` using `StoreId`</RN>
<RN type="feature" platform="iOS">DRM-aware ‚Äî `ContentUri` is empty for Apple Music subscription tracks that cannot be copied, but streaming playback is available via `StoreId`</RN>
<RN type="feature" platform="iOS">`IsExplicit` property on `MusicMetadata` ‚Äî reports whether a track is marked as explicit content via `MPMediaItem.IsExplicitItem`</RN>
