## 2.0 - March 22, 2026
<RN type="breaking">DI extensions moved to separate `Shiny.SqliteDocumentDb.Extensions.DependencyInjection` package тАФ the core library no longer depends on `Microsoft.Extensions.DependencyInjection.Abstractions`</RN>
<RN type="feature">Convenience constructor тАФ `new SqliteDocumentStore("Data Source=mydata.db")` for quick setup without options</RN>
<RN type="feature">Configurable default table name via `DocumentStoreOptions.TableName` (defaults to `"documents"`)</RN>
<RN type="feature">Table-per-type mapping тАФ `MapTypeToTable<T>()` gives a document type its own dedicated SQLite table with lazy creation on first use</RN>
<RN type="feature">Auto-derived or explicit table names тАФ `MapTypeToTable<T>()` derives from the type name, `MapTypeToTable<T>(string)` uses an explicit name</RN>
<RN type="feature">Duplicate table name protection тАФ mapping two types to the same custom table throws `InvalidOperationException`</RN>
<RN type="feature">Custom Id property mapping тАФ `MapTypeToTable<T>("table", x => x.MyProperty)` uses an alternate property as the document Id instead of the default `Id`</RN>
<RN type="feature">Fluent options API тАФ all `MapTypeToTable` overloads return `DocumentStoreOptions` for chaining</RN>
<RN type="feature">Document diffing via `GetDiff<T>(id, modified)` тАФ compares a modified object against the stored document and returns an RFC 6902 `JsonPatchDocument<T>` with deep nested-object diffing powered by SystemTextJsonPatch</RN>
<RN type="feature">All new features are fully AOT-safe тАФ type names and Id property names are resolved at registration time, not at runtime</RN>
<RN type="feature">Batch insert via `BatchInsert<T>(IEnumerable<T>)` тАФ inserts a collection in a single transaction with prepared command reuse, auto-generates IDs, and rolls back atomically on failure</RN>

## 1.0 - March 6, 2026
<RN type="feature">Schema-free JSON document storage on top of SQLite</RN>
<RN type="feature">Mandatory typed `Id` property on document types (`Guid`, `int`, `long`, or `string`) тАФ stored in both the SQLite column and the JSON blob so query results always include it</RN>
<RN type="feature">Auto-generation of Ids on Insert: `Guid` тЖТ `Guid.NewGuid()`, `int`/`long` тЖТ `MAX(CAST(Id AS INTEGER)) + 1` per TypeName. String Ids must be set explicitly тАФ Insert throws for default string Ids</RN>
<RN type="feature">LINQ expression queries translated to json_extract SQL with support for equality, comparisons, logical operators, null checks, string methods, nested properties, and collection queries</RN>
<RN type="feature">Fluent query builder (IDocumentQuery) тАФ chain .Where(), .OrderBy(), .OrderByDescending(), .GroupBy(), .Paginate(), .Select() and terminate with .ToList(), .ToAsyncEnumerable(), .Count(), .Any(), .ExecuteDelete(), .ExecuteUpdate(), .Max(), .Min(), .Sum(), .Average()</RN>
<RN type="feature">Pagination via .Paginate(offset, take) тАФ translates to SQL LIMIT/OFFSET</RN>
<RN type="feature">Expression-based ordering тАФ .OrderBy(u => u.Age) and .OrderByDescending(u => u.Name) on the fluent query builder</RN>
<RN type="feature">SQL-level projections via .Select() using json_object for extracting specific fields without full deserialization</RN>
<RN type="feature">IAsyncEnumerable streaming via .ToAsyncEnumerable() тАФ yield results one-at-a-time without buffering</RN>
<RN type="feature">Expression-based JSON indexes for up to 30x faster queries on indexed properties</RN>
<RN type="feature">Full AOT and trimming support тАФ all JsonTypeInfo parameters are optional and auto-resolve from configured JsonSerializerContext</RN>
<RN type="feature">Scalar aggregates: .Max(), .Min(), .Sum(), .Average() as terminal methods on the query builder</RN>
<RN type="feature">Aggregate projections with automatic GROUP BY via Sql.Count(), Sql.Max(), Sql.Min(), Sql.Sum(), Sql.Avg() marker methods</RN>
<RN type="feature">Collection-level aggregates in projections: Sum, Min, Max, Average on child collections (e.g. o.Lines.Sum(l => l.Quantity))</RN>
<RN type="feature">Explicit Insert / Update / Upsert API тАФ Insert throws on duplicate Ids, Update throws if not found, Upsert deep-merges via json_patch</RN>
<RN type="feature">SetProperty тАФ update a single scalar JSON field via json_set without deserializing the document. Supports nested paths</RN>
<RN type="feature">RemoveProperty тАФ strip a field from the stored JSON via json_remove. Works on any property type</RN>
<RN type="feature">Typed Id lookups тАФ Get, Remove, SetProperty, and RemoveProperty accept the Id as `object` (Guid, int, long, or string). Unsupported types throw ArgumentException</RN>
<RN type="feature">Bulk delete via query builder тАФ .Where(predicate).ExecuteDelete() returns count of deleted documents</RN>
<RN type="feature">Bulk update via query builder тАФ .Where(predicate).ExecuteUpdate(property, value) updates a property on matching documents via json_set() and returns count updated</RN>
<RN type="feature">Transactions with automatic commit/rollback via RunInTransaction</RN>
<RN type="feature">Hot backup via store.Backup(path) тАФ copies the database to a file using the SQLite Online Backup API while the store remains usable</RN>
<RN type="feature">Dependency injection registration via AddSqliteDocumentStore</RN>
<RN type="feature">Configurable type name resolution (ShortName or FullName)</RN>
<RN type="feature">UseReflectionFallback option for strict AOT enforcement</RN>
<RN type="feature">SQL logging callback via DocumentStoreOptions.Logging</RN>
<RN type="feature">Raw SQL query and streaming support via store.Query(whereClause) and store.QueryStream(whereClause)</RN>
