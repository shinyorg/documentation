## v4

### 4.1.0 - April 6, 2026
<RN type="feature">Stationary detection now built into all GPS providers. `GpsReading.IsStationary` is set automatically on Android, iOS legacy, and iOS 18+ before readings reach delegates or the observable stream.</RN>
<RN type="enhancement" platform="Android">`AndroidGpsRequest` now exposes `StationaryMetersThreshold` and `StationarySecondsThreshold` to configure stationary detection sensitivity.</RN>
<RN type="enhancement" platform="iOS">`AppleGpsRequest` now exposes `StationaryMetersThreshold` and `StationarySecondsThreshold` for legacy iOS stationary detection. iOS 18+ continues to use native `CLLocationUpdater` stationary detection.</RN>
<RN type="chore" breaking>Stationary detection removed from `GpsDelegate`. The `IsStationary`, `DetectStationary`, `StationaryMetersThreshold`, and `StationarySecondsThreshold` properties have been removed. Use `GpsReading.IsStationary` instead.</RN>
<RN type="feature" platform="Blazor">New `Shiny.Locations.Blazor` package brings foreground GPS to Blazor WebAssembly via `navigator.geolocation`. Register with `builder.Services.AddGps()` and inject `IGpsManager` like any other platform. Background GPS and geofencing are not supported by browsers and remain unavailable on web.</RN>

### 4.0.0 - March 26, 2026
<RN type="feature">Windows support added (No Background Support at this time)</RN>
<RN type="fix" platform="Android">Don't request ACCESS_BACKGROUND_LOCATION unless realtime GPS request and less than API level 31 OR standard location background tracking is being requested</RN>
<RN type="fix">The base GpsDelegate calculations could receive a batch and trigger multiple calculations.  This has been made into a synchronized operation</RN>
<RN type="fix" platform="iOS" githubNumber={1544}>Geofence Manager RequestState works on new CLMonitor API</RN>
<RN type="enhancement" platform="iOS" githubNumber={1396}>GPS background will now request background permission right away</RN>
<RN type="enhancement">GpsDelegate now has a boolean to detect if stationary</RN>
<RN type="enhancement" platform="iOS">iOS 18+ now uses CLMonitor for GPS</RN>
<RN type="enhancement">New geofence registration mechanics for iOS 17+</RN>
