## 1.0 - May 2026
<RN type="feature">Initial release of Shiny.DataSync — AOT-compliant offline-first data synchronization for .NET MAUI and desktop apps</RN>
<RN type="feature">`ISyncService` with full CRUD operations: `Insert`, `Update`, `Remove`, typed and untyped variants</RN>
<RN type="feature">`Sync()` runs a full cycle: Clean → Push → Pull with per-entity type granularity</RN>
<RN type="feature">Per-entity sync direction: `SyncDirection.Both`, `PullOnly`, and `PushOnly`</RN>
<RN type="feature">Configurable pull throttling via `PullMinimumTime` and incremental pull via `PullDateVariable`</RN>
<RN type="feature">Batched push with configurable `PushBufferSize` (default: 50 entities per request)</RN>
<RN type="feature">Four composable data removal strategies: soft-delete, tombstone, reconciliation, and expiry predicates</RN>
<RN type="feature">Reconciliation safety — skips entities with pending local changes to avoid removing in-progress work</RN>
<RN type="feature">`ISyncInterceptor` for per-request HTTP modifications (auth headers, custom headers)</RN>
<RN type="feature">Observable sync events via `WhenSync()` — push/pull started, completed, and failed with entity type and count</RN>
<RN type="feature">Push queue management: `GetQueue()`, `Clear()`, and `GetMetadata()` with per-entity push attempt tracking</RN>
<RN type="feature">`MaxPushAttempts` configuration — automatically drops poison queue items after N failed attempts during Clean</RN>
<RN type="feature">Automatic Shiny Jobs background sync registration — no manual `AddJob()` needed</RN>
<RN type="feature">Separate `DeleteUri` support for APIs that use different endpoints for create/update vs. delete</RN>
<RN type="feature">Version tracking via `VersionSelector` for server-driven versioning</RN>
<RN type="feature">Built on Shiny.DocumentDb.Sqlite for local entity storage and internal metadata</RN>
<RN type="feature">Full AOT compliance — all serialization uses `System.Text.Json` source-generated contexts</RN>
