## v1.0.0 - TBD
<RN type="feature">Command-object pattern with generic return types (`IObdCommand<T>`, `ObdCommand<T>`) for compile-time safe OBD communication</RN>
<RN type="feature">9 standard OBD-II commands: Vehicle Speed, Engine RPM, Coolant Temperature, Throttle Position, Fuel Level, Calculated Engine Load, Intake Air Temperature, Runtime Since Start, VIN</RN>
<RN type="feature">ELM327 protocol handling with hex response parsing, multi-frame CAN support, and automatic error detection</RN>
<RN type="feature">Adapter auto-detection — probes via ATI to identify ELM327 vs OBDLink (STN) adapters and selects the appropriate initialization sequence</RN>
<RN type="feature">Adapter profile system (`IObdAdapterProfile`) with built-in `Elm327AdapterProfile` and `ObdLinkAdapterProfile`</RN>
<RN type="feature">Device discovery via `IObdDeviceScanner` and `ObdDiscoveredDevice` for scanning available adapters before connecting</RN>
<RN type="feature">Pluggable transport abstraction (`IObdTransport`) for BLE, WiFi, USB, or any communication channel</RN>
<RN type="feature">BLE transport (`BleObdTransport`) using Shiny.BluetoothLE v4 with auto-scan, pre-discovered peripheral, or discovered device constructors</RN>
<RN type="feature">BLE device scanner (`BleObdDeviceScanner`) with optional name filtering and UUID-based deduplication</RN>
<RN type="feature">Configurable BLE defaults (`BleObdConfiguration`) for GATT UUIDs, device name filter, and command timeout</RN>
<RN type="feature">DI extension method `AddShinyObdBluetoothLE()` for one-call BLE OBD service registration</RN>
<RN type="feature">Fully task-based async throughout — no Reactive Extensions required in consuming code</RN>
