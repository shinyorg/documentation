## v1

### 1.3 - March 13, 2026
<RN type="feature">`GetPlaylistsAsync()` on `IMediaLibrary` тАФ returns all playlists from the device music library with song counts, sorted alphabetically</RN>
<RN type="feature">`GetPlaylistTracksAsync(playlistId)` on `IMediaLibrary` тАФ returns all tracks in a playlist, in playlist order</RN>
<RN type="feature">`PlaylistInfo` record тАФ contains playlist Id, Name, and SongCount</RN>
<RN type="feature" platform="Android">Android playlist querying via `MediaStore.Audio.Playlists` and `MediaStore.Audio.Playlists.Members`</RN>
<RN type="feature" platform="iOS">iOS playlist querying via `MPMediaQuery.PlaylistsQuery` and `MPMediaPlaylist`</RN>

### 1.2 - March 13, 2026
<RN type="feature">`MusicFilter` class тАФ combine Genre, Year, Decade, and SearchQuery criteria with AND logic to filter tracks</RN>
<RN type="feature">`GetTracksAsync(MusicFilter)` on `IMediaLibrary` тАФ query tracks matching any combination of filter criteria</RN>
<RN type="feature">`GetYearsAsync()` on `IMediaLibrary` тАФ returns distinct release years with track counts, with optional `MusicFilter` parameter</RN>
<RN type="feature">`GetDecadesAsync()` on `IMediaLibrary` тАФ returns distinct decades (e.g., 1990s) with track counts, with optional `MusicFilter` parameter</RN>
<RN type="feature">`GroupedCount<T>` record тАФ generic type pairing a grouped value with its track count, used by all grouping methods</RN>
<RN type="feature">`Year` property on `MusicMetadata` тАФ the release year of the track (nullable)</RN>
<RN type="feature">`GetGenresAsync` now accepts an optional `MusicFilter` and returns `GroupedCount<string>` with track counts</RN>
<RN type="feature">Cross-dimensional browsing тАФ get genres by decade, decades by genre, years by genre, etc.</RN>
<RN type="feature" platform="Android">Year metadata read from `MediaStore.Audio.Media.YEAR`</RN>
<RN type="feature" platform="iOS">Year metadata extracted from `MPMediaItem.ReleaseDate`</RN>
