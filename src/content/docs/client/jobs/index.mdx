Shiny Jobs allow you to run periodic background tasks. On Android, this is powered by WorkManager. On iOS, by BGTaskScheduler. On Blazor WebAssembly, jobs run in-process while the tab is open, with state persisted to browser localStorage via `Shiny.Support.Storage.Blazor`.

:::caution
Jobs are **not** scheduled at specific times. The operating system determines when to run them based on device conditions, battery, and usage patterns.
:::

:::warning
Jobs do not work on the iOS Simulator. You must test on a real device.
:::

## Platform Notes

| Platform | Package | Notes |
|----------|---------|-------|
| Android | `Shiny.Jobs` | WorkManager — true OS-managed background execution |
| iOS | `Shiny.Jobs` | BGTaskScheduler — OS decides when |
| Blazor WebAssembly | `Shiny.Jobs.Blazor` | In-process scheduler, persisted to browser localStorage |

:::caution[macOS, Windows, Linux]
There is **no native Shiny Jobs implementation** for macOS, Windows, or Linux. These desktop platforms don't aggressively suspend processes the way mobile OSes do, so a background-job abstraction is unnecessary — run a `Timer`, `BackgroundService`, or hosted service directly.
:::

:::caution[Blazor WebAssembly]
The Blazor implementation is **not a true background job runner** — it can only execute while the browser tab is alive. Jobs and their state survive page reloads (via localStorage), but nothing runs while the tab is closed. For "wake the user when the app is closed" scenarios, use [Push Notifications](/client/push) instead.
:::

## Features

- Cross-platform background job execution
- Runtime criteria (internet access, charging, battery level)
- Full dependency injection support
- Stateful jobs (persists state across runs)
- Foreground execution option
