## Features

- **Navigation service** тАФ route-based and ViewModel-based navigation with `INavigator`
- **XAML navigation** тАФ attached `Navigate.*` properties for `Button`, `MenuItem`, and `ToolbarItem`
- **Tab badges** тАФ set and clear numeric tab badges by route or ViewModel
- **Shell switching** тАФ swap the entire Shell at runtime with `SwitchShell` (instance or DI-resolved)
- **Dialog service** тАФ alert, confirm, prompt, and action sheet dialogs with `IDialogs`
- **Navigation events** тАФ `Navigating` and `Navigated` events with source/destination ViewModel instances
- **ViewModel lifecycle** тАФ appearing/disappearing, navigation confirmation, navigation awareness, and disposal
- **Auto BindingContext** тАФ ViewModels automatically attached to page binding contexts
- **Source generation** тАФ eliminates route registration boilerplate, generates strongly-typed navigation extensions and DI registration
- **AI integration** тАФ source-generated route metadata and navigation tools for [Microsoft.Extensions.AI](https://www.nuget.org/packages/Microsoft.Extensions.AI) тАФ let an AI chat agent discover pages, match user intent, and navigate with pre-filled forms
- **ShinyShell base class** тАФ one base class change for deterministic BindingContext assignment on the initial page
- **Testable** тАФ `INavigator` and `IDialogs` are injectable and mockable for unit testing

## Setup

<Steps>
1. Install the NuGet package

   <NugetBadge name="Shiny.Maui.Shell" showLabel={true} />

   ```bash
   dotnet add package Shiny.Maui.Shell
   ```

2. Configure `UseShinyShell` in your `MauiProgram.cs`

   ```csharp
   public static class MauiProgram
   {
       public static MauiApp CreateMauiApp()
       {
           var builder = MauiApp.CreateBuilder();
           builder
               .UseMauiApp<App>()
               .UseShinyShell(x => x
                   .Add<MainPage, MainViewModel>(registerRoute: false)
                   .Add<AnotherPage, AnotherViewModel>("another")
               )
               .ConfigureFonts(fonts =>
               {
                   fonts.AddFont("OpenSans-Regular.ttf", "OpenSansRegular");
                   fonts.AddFont("OpenSans-Semibold.ttf", "OpenSansSemibold");
               });

           return builder.Build();
       }
   }
   ```

3. Update your `AppShell` to inherit from `ShinyShell`

   **AppShell.xaml:**
   ```xml
   <shiny:ShinyShell
       x:Class="MyApp.AppShell"
       xmlns="http://schemas.microsoft.com/dotnet/2021/maui"
       xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
       xmlns:shiny="clr-namespace:Shiny;assembly=Shiny.Maui.Shell"
       xmlns:local="clr-namespace:MyApp"
       Title="MyApp">

       <ShellContent
           Title="Home"
           ContentTemplate="{DataTemplate local:MainPage}"
           Route="MainPage" />

   </shiny:ShinyShell>
   ```

   **AppShell.xaml.cs:**
   ```csharp
   using Shiny;

   namespace MyApp;

   public partial class AppShell : ShinyShell
   {
       public AppShell()
       {
           InitializeComponent();
       }
   }
   ```

4. Inject `INavigator` into your ViewModels and navigate

   ```csharp
   using Shiny;

   public class MainViewModel(INavigator navigator)
   {
       public async Task GoToDetails()
       {
           await navigator.NavigateTo("another");
       }
   }
   ```
</Steps>

:::note
Your AppShell must inherit from `ShinyShell` тАФ this ensures the initial page's BindingContext is set deterministically via Shell's own `OnNavigated` lifecycle.
:::

## Quick Example

```csharp
// Navigate by route with arguments
await navigator.NavigateTo("details", ("Id", 42));

// Navigate by ViewModel with strongly typed setup
await navigator.NavigateTo<DetailViewModel>(vm => vm.Id = 42);

// Set a numeric badge on a tab
await navigator.SetTabBadge("Inbox", 3);

// Go back with arguments
await navigator.GoBack(("Result", "saved"));

// Switch to a different Shell at runtime
await navigator.SwitchShell<AdminShell>();
```

## XAML Navigation

`Navigate` attached properties let you trigger route-based navigation directly from XAML without creating a ViewModel command first.

```xml
<ContentPage
    xmlns="http://schemas.microsoft.com/dotnet/2021/maui"
    xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
    xmlns:shiny="clr-namespace:Shiny;assembly=Shiny.Maui.Shell">

    <Button Text="Open Detail"
            shiny:Navigate.Route="Detail"
            shiny:Navigate.ParameterKey="ItemId"
            shiny:Navigate.ParameterValue="{Binding SelectedId}" />

    <ToolbarItem Text="Home"
                 shiny:Navigate.Route="MainPage"
                 shiny:Navigate.RelativeNavigation="False" />
</ContentPage>
```

`Navigate` currently supports `Button`, `MenuItem`, and `ToolbarItem`.

## Tab Badges

Use `INavigator` to update numeric tab badges for routes that already exist in the active Shell:

```csharp
await navigator.SetTabBadge("Inbox", 3);
await navigator.SetTabBadge<InboxViewModel>(7);
await navigator.ClearTabBadge("Inbox");
await navigator.ClearTabBadge<InboxViewModel>();
```

Tab badges are supported on Android, iOS, Mac Catalyst, and Windows. Linux and macOS AppKit throw `PlatformNotSupportedException`.
