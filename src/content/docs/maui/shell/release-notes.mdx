## v3.0 - March 11, 2026

<RN type="feature">**IDialogs interface** ‚Äî new dedicated dialog service with `Alert`, `Confirm`, `Prompt`, and `ActionSheet` methods, injected separately from `INavigator` for clean separation of concerns</RN>
<RN type="feature">`Prompt` dialog ‚Äî display a text input dialog with customizable keyboard type, placeholder, initial value, and max length</RN>
<RN type="feature" githubNumber="1" pr={true} contributor="b4j4" repo="mauishell">`ActionSheet` dialog ‚Äî display a multi-option action sheet with cancel and destructive action support</RN>
<RN type="enhancement" breaking={true}>`Alert` and `Confirm` moved from `INavigator` to `IDialogs` ‚Äî improves testability and follows interface segregation</RN>

### Migration from v2.x

- `INavigator.Alert()` and `INavigator.Confirm()` have been removed ‚Äî inject `IDialogs` instead
- Replace `navigator.Alert(...)` with `dialogs.Alert(...)` and `navigator.Confirm(...)` with `dialogs.Confirm(...)`
- Add `IDialogs` to your ViewModel constructor parameters where dialogs are used
- `IDialogs` is automatically registered by `UseShinyShell()` ‚Äî no additional setup required

```csharp
// Before (v2.x)
public class MyViewModel(INavigator navigator)
{
    await navigator.Alert("Error", "Something went wrong");
    bool ok = await navigator.Confirm("Delete?", "Are you sure?");
}

// After (v3.0)
public class MyViewModel(INavigator navigator, IDialogs dialogs)
{
    await dialogs.Alert("Error", "Something went wrong");
    bool ok = await dialogs.Confirm("Delete?", "Are you sure?");

    // New capabilities
    var name = await dialogs.Prompt("Name", "Enter your name");
    var choice = await dialogs.ActionSheet("Options", "Cancel", null, "Edit", "Share");
}
```

## v2.2

<RN type="feature">Configurable source generation ‚Äî disable route constants via `ShinyMauiShell_GenerateRouteConstants` or navigation extensions via `ShinyMauiShell_GenerateNavExtensions` MSBuild properties (empty/missing = enabled, `false` = disabled)</RN>
<RN type="feature">Route-based naming ‚Äî the `route` parameter in `[ShellMap]` now drives the generated constant name and navigation method name (e.g., `[ShellMap<HomePage>("Dashboard")]` ‚Üí `Routes.Dashboard`, `NavigateToDashboard`)</RN>
<RN type="feature">Invalid route diagnostic ‚Äî **SHINY001** compiler error when the route value is not a valid C# identifier (hyphens, spaces, leading digits)</RN>
<RN type="enhancement">`AddGeneratedMaps()` is always generated ‚Äî even before any `[ShellMap]` attributes exist ‚Äî so you can wire up `MauiProgram.cs` immediately</RN>
<RN type="enhancement">`AddGeneratedMaps()` now uses inline string literals instead of `Routes.*` constants, so it works correctly even when route constant generation is disabled</RN>
<RN type="enhancement">When no route is specified, the generated name falls back to the page type name without the `Page` suffix (e.g., `[ShellMap<HomePage>]` ‚Üí `Routes.Home`)</RN>

### Migration from v2.1

- Route constant names may change if you specified explicit routes ‚Äî e.g., `Routes.Home` for `[ShellMap<HomePage>("Dashboard")]` is now `Routes.Dashboard`
- Navigation extension method names change similarly ‚Äî `NavigateToHome` becomes `NavigateToDashboard`
- Routes with invalid C# identifiers (hyphens, spaces, leading digits) now produce compile errors ‚Äî rename them to valid identifiers
