## v1

### 1.2 - May 6, 2026
<RN type="feature" platform="WASM">Browser `IAudioSource` implementation — raw PCM microphone capture via the Web Audio API (`getUserMedia` + `ScriptProcessorNode`), downsampled to 16kHz 16-bit mono, matching the output format of Android, iOS, and Windows</RN>
<RN type="enhancement" platform="WASM">Cloud STT providers (Azure, custom) now work in the browser — `IAudioSource` provides the raw audio stream that `CloudSpeechToText` requires</RN>

### 1.1.2 - May 6, 2026
<RN type="enhancement">Cloud provider extensions (`AddAzureSpeech`, `AddElevenLabsTextToSpeech`, `AddCloudSpeechToText`, `AddCloudTextToSpeech`) now automatically register `IAudioSource` and `IAudioPlayer` — manual `AddAudioSource()` / `AddAudioPlayer()` calls are no longer required</RN>

### 1.1.1 - May 5, 2026
<RN type="feature">`ISpeechToTextService.IsListening` property — indicates whether speech recognition is currently active, analogous to `ITextToSpeechService.IsSpeaking`</RN>

### 1.1 - May 4, 2026
<RN type="feature">`ListenWithWakeWord()` extension method — "Hey Siri" style wake word activation that continuously listens for a wake phrase, then captures the spoken command after it until silence</RN>
<RN type="feature">`ListenForKeyword()` extension method — listens continuously until one of the specified keywords is detected (case-insensitive, whole-word matching), returns the matched keyword</RN>
<RN type="feature">Wake word supports pause-then-speak — if the user says the wake phrase and pauses before speaking, the method waits for the next utterance as the command</RN>
<RN type="feature">Both methods are extension methods on `ISpeechToTextService` composing over `ContinuousRecognize` — no platform-specific code changes required</RN>
<RN type="feature">Sample apps updated with Wake Word and Keyword listening modes (MAUI + Blazor)</RN>

### 1.0 - May 2, 2026
<RN type="feature">`ISpeechToTextService` interface — platform-native speech recognition with permission management, continuous streaming, and listen-until-silence modes</RN>
<RN type="feature">`ITextToSpeechService` interface — platform-native text-to-speech with voice selection, speech rate, pitch, and volume control</RN>
<RN type="feature">`IAudioSource` interface — raw PCM audio capture from the device microphone (16kHz, 16-bit, mono)</RN>
<RN type="feature">`IAudioPlayer` interface — MP3 audio stream playback with play/stop control</RN>
<RN type="feature">`SpeechRecognitionOptions` — configurable culture, silence timeout, and on-device preference for STT</RN>
<RN type="feature">`TextToSpeechOptions` — configurable culture, voice, speech rate, pitch, and volume for TTS</RN>
<RN type="feature">`ContinuousRecognize()` — streaming recognition results via `IAsyncEnumerable<SpeechRecognitionResult>` with partial and final results</RN>
<RN type="feature">`ListenUntilSilence()` — simple dictation mode that returns the final transcription after silence is detected</RN>
<RN type="feature">`GetVoicesAsync()` — enumerate available TTS voices with optional culture filtering</RN>
<RN type="feature">`AddSpeechServices()` — single extension method to register all core services (STT, TTS, AudioSource, AudioPlayer)</RN>
<RN type="feature" platform="Android">Android STT implementation using `SpeechRecognizer` with streaming partial results</RN>
<RN type="feature" platform="Android">Android TTS implementation using `Android.Speech.Tts.TextToSpeech`</RN>
<RN type="feature" platform="Android">Android audio capture via `AudioRecord` with 16kHz PCM streaming</RN>
<RN type="feature" platform="Android">Android audio playback via `MediaPlayer`</RN>
<RN type="feature" platform="iOS">iOS STT implementation using `SFSpeechRecognizer` with `SFSpeechAudioBufferRecognitionRequest`</RN>
<RN type="feature" platform="iOS">iOS TTS implementation using `AVSpeechSynthesizer`</RN>
<RN type="feature" platform="iOS">iOS audio capture via `AVAudioEngine` with PCM tap</RN>
<RN type="feature" platform="iOS">iOS audio playback via `AVAudioPlayer`</RN>
<RN type="feature">Cloud provider abstraction — `ISpeechToTextProvider` and `ITextToSpeechProvider` interfaces for pluggable cloud backends</RN>
<RN type="feature">`CloudSpeechToText` and `CloudTextToSpeech` — bridge classes that combine platform audio with cloud provider APIs</RN>
<RN type="feature">`AddCloudSpeechToText<T>()` and `AddCloudTextToSpeech<T>()` — generic DI registration for custom cloud providers</RN>
<RN type="feature">Azure AI Speech provider — `AddAzureSpeech()` registers Azure STT and/or TTS with subscription key and region</RN>
<RN type="feature">Azure TTS with SSML prosody control — speech rate, pitch, and volume mapped to SSML elements</RN>
<RN type="feature">ElevenLabs TTS provider — `AddElevenLabsTextToSpeech()` registers ElevenLabs cloud TTS with configurable voice and model</RN>
<RN type="feature">`PipeStream` utility — thread-safe producer-consumer stream using `System.IO.Pipelines` for bridging audio capture with cloud providers</RN>
<RN type="feature" platform="WASM">Browser/WebAssembly support — STT and TTS via Web Speech API, auto-detected at runtime via `OperatingSystem.IsBrowser()`</RN>
<RN type="feature" platform="WASM">Browser STT implementation using `SpeechRecognition` API with streaming partial and final results</RN>
<RN type="feature" platform="WASM">Browser TTS implementation using `SpeechSynthesis` API with voice selection, rate, pitch, and volume control</RN>
<RN type="feature" platform="WASM">Browser audio playback via HTML5 `Audio` element with base64 data URL conversion</RN>
<RN type="feature">Blazor WebAssembly sample app demonstrating STT, TTS, and voice listing</RN>
