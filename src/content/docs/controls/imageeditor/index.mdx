An inline image editor with cropping (drag-handle selection with dimmed overlay), rotation, freehand drawing with color, text annotations, undo/redo, reset-to-original, and export to PNG/JPEG/WEBP at configurable resolutions. Every feature can be toggled on or off, and the default toolbar can be replaced with a custom template.

## Basic Usage

```xml
<shiny:ImageEditor Source="{Binding ImageData}"
                   CurrentToolMode="{Binding ToolMode}"
                   AllowCrop="True"
                   AllowRotate="True"
                   AllowDraw="True"
                   AllowTextAnnotation="True"
                   DrawStrokeColor="Red"
                   DrawStrokeWidth="3" />
```

## Properties

| Property | Type | Default | Description |
|---|---|---|---|
| `Source` | `byte[]?` | `null` | Image data to edit |
| `CurrentToolMode` | `ImageEditorToolMode` | `None` | Active tool: None, Crop, Draw, Text (TwoWay bindable) |
| `AllowCrop` | `bool` | `true` | Enable/disable crop tool |
| `AllowRotate` | `bool` | `true` | Enable/disable rotate action |
| `AllowDraw` | `bool` | `true` | Enable/disable freehand drawing |
| `AllowTextAnnotation` | `bool` | `true` | Enable/disable text annotations |
| `AllowZoom` | `bool` | `true` | Enable/disable pinch-to-zoom |
| `CanUndo` | `bool` | `false` | Whether undo is available (OneWayToSource) |
| `CanRedo` | `bool` | `false` | Whether redo is available (OneWayToSource) |
| `DrawStrokeColor` | `Color` | `Red` | Drawing stroke color |
| `DrawStrokeWidth` | `double` | `3` | Drawing stroke width |
| `TextFontSize` | `double` | `16` | Text annotation font size |
| `AnnotationTextColor` | `Color` | `White` | Text annotation color |
| `ToolbarTemplate` | `DataTemplate?` | `null` | Custom toolbar (replaces the default toolbar) |
| `ToolbarPosition` | `ToolbarPosition` | `Bottom` | Toolbar placement: Top or Bottom |
| `UseHapticFeedback` | `bool` | `true` | Haptic feedback on actions (MAUI only) |

## Commands

All editing actions are exposed as `ICommand` properties for use with custom toolbars or ViewModels:

| Command | Parameter | Description |
|---|---|---|
| `UndoCommand` | -- | Undo the last edit action |
| `RedoCommand` | -- | Redo the last undone action |
| `RotateCommand` | `float` (degrees) | Rotate the image |
| `ResetCommand` | -- | Clear all edits and restore the original image |
| `CropCommand` | -- | Toggle crop mode on/off |
| `DrawCommand` | -- | Toggle draw mode on/off |
| `TextCommand` | -- | Toggle text annotation mode on/off |

## Methods

| Method | Returns | Description |
|---|---|---|
| `Undo()` | `void` | Undo the last action |
| `Redo()` | `void` | Redo the last undone action |
| `Rotate(float degrees)` | `void` | Rotate by the given angle |
| `Reset()` | `void` | Clear all edits |
| `ApplyCrop()` | `void` | Commit the active crop selection |
| `ExportAsync(ImageExportOptions?)` | `Task<Stream>` | Export the edited image to a stream |

## Features

- **Crop** -- Drag-handle area selection that starts with the full image selected. Areas outside the crop are dimmed with a semi-transparent overlay; the selected area stays fully lit. 8 drag handles (4 corners + 4 midpoints) with a rule-of-thirds grid overlay. Confirm or cancel from the toolbar.
- **Rotate** -- 90-degree increments or arbitrary angles. Each rotation is a separate undoable action.
- **Freehand drawing** -- Draw strokes on the image with configurable color and width. Each completed stroke is one undoable action.
- **Text annotations** -- Tap the image to place text. Prompts for input with configurable font size and color.
- **Pinch-to-zoom** -- View-only zoom for inspecting details. Does not affect the exported image.
- **Undo/redo** -- Every edit action is pushed to a stack. Undo moves the last action to a redo stack; redo re-applies it. New actions clear the redo stack.
- **Reset** -- Clears all actions and restores the original image.
- **Export** -- Render the edited image to a stream at a target format (PNG, JPEG, WEBP) and resolution.
- **Toolbar** -- Default toolbar shows buttons for each enabled feature, plus undo/redo/reset. Setting `AllowX=false` hides the corresponding button. Replace the entire toolbar with `ToolbarTemplate` for a fully custom UI while still driving actions via the exposed commands and methods.

## ViewModel Pattern

```csharp
public partial class ImageEditorViewModel : ObservableObject
{
    [ObservableProperty] byte[]? imageData;
    [ObservableProperty] bool canUndo;
    [ObservableProperty] bool canRedo;
    [ObservableProperty] ImageEditorToolMode currentToolMode;
    [ObservableProperty] Color drawColor = Colors.Red;

    [RelayCommand]
    async Task LoadImage()
    {
        var result = await MediaPicker.PickPhotoAsync();
        if (result == null) return;
        using var stream = await result.OpenReadAsync();
        using var ms = new MemoryStream();
        await stream.CopyToAsync(ms);
        ImageData = ms.ToArray();
    }
}
```

:::tip
`Source` accepts `byte[]`, not `ImageSource`. Load image data from a file, stream, or MediaPicker result and assign the byte array.
:::

## Export

Export the edited image at a specific format and resolution:

```csharp
var stream = await editor.ExportAsync(new ImageExportOptions
{
    Format = ImageExportFormat.Jpeg,
    Quality = 0.85f,
    Width = 1920,
    Height = 1080
});
```

`ImageExportOptions` properties:

| Property | Type | Default | Description |
|---|---|---|---|
| `Format` | `ImageExportFormat` | `Png` | Output format: Png, Jpeg, or Webp |
| `Quality` | `float` | `0.92` | Compression quality (0--1, applies to Jpeg/Webp) |
| `Width` | `int?` | `null` | Target width (null = original image width) |
| `Height` | `int?` | `null` | Target height (null = original image height) |

## Blazor Usage

The Blazor `<ImageEditor>` component mirrors the MAUI control. Use `@ref` to call methods like `ExportAsync`:

```razor
<ImageEditor @ref="editor"
             Source="@imageUrl"
             ImageData="@imageData"
             AllowCrop="true"
             AllowDraw="true"
             DrawStrokeColor="#ff0000"
             CanUndoChanged="v => canUndo = v"
             CanRedoChanged="v => canRedo = v" />

@code {
    ImageEditor? editor;
    string? imageUrl = "https://example.com/photo.jpg";
    byte[]? imageData;
    bool canUndo, canRedo;

    async Task Export()
    {
        var bytes = await editor!.ExportAsync("png");
    }
}
```

:::note
On Blazor, `Source` accepts a URL string and `ImageData` accepts `byte[]`. Colors are CSS color strings (e.g. `"#ff0000"`). `ExportAsync` returns `byte[]` instead of `Stream`.
:::
