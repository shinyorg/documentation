Every interactive control in the library fires feedback events through `IFeedbackService` — a single, injectable service that lets you customize how your app responds to user interactions. By default, it triggers haptic feedback. Replace it with text-to-speech, sound effects, analytics, or any combination.

## How It Works

All Shiny controls that support feedback have a `UseFeedback` property (default: `true`). When an interaction occurs — a button tap, a pin digit entered, a toast shown, a message received — the control calls `IFeedbackService.OnRequested()` with:

| Parameter | Description |
|---|---|
| `controlType` | The `Type` of the control (e.g. `typeof(ChatView)`, `typeof(SecurityPin)`) |
| `eventName` | The interaction that occurred (e.g. `"MessageSent"`, `"Completed"`, `"Clicked"`) |
| `details` | Optional context — for `ChatView`, this is the message text; for `SecurityPin`, `"LongPress"` on completion |

## Default Behavior

`UseShinyControls()` registers `HapticFeedbackService` as the default implementation. It performs a click haptic for most events and a long press haptic when `details` is `"LongPress"`.

```csharp
builder.UseShinyControls(); // registers HapticFeedbackService by default
```

## Custom Feedback Service

Replace the default by calling `SetCustomFeedback<T>()` during setup:

```csharp
builder.UseShinyControls(cfg =>
{
    cfg.SetCustomFeedback<MyFeedbackService>();
});
```

### Example: Text-to-Speech for Chat

```csharp
public class ChatTtsFeedbackService : IFeedbackService
{
    readonly ITextToSpeech tts;
    public ChatTtsFeedbackService(ITextToSpeech tts) => this.tts = tts;

    public async void OnRequested(Type controlType, string eventName, string? details)
    {
        // Speak incoming chat messages aloud
        if (controlType == typeof(ChatView) && eventName == "MessageReceived" && details is not null)
        {
            await tts.SpeakAsync(details);
            return;
        }

        // Default haptic for everything else
        try
        {
            HapticFeedback.Default.Perform(HapticFeedbackType.Click);
        }
        catch { }
    }
}
```

### Example: Sound Effects

```csharp
public class SoundFeedbackService : IFeedbackService
{
    public void OnRequested(Type controlType, string eventName, string? details)
    {
        var sound = (controlType.Name, eventName) switch
        {
            ("SecurityPin", "Completed") => "success.wav",
            ("SecurityPin", "DigitEntered") => "click.wav",
            ("ChatView", "MessageSent") => "swoosh.wav",
            ("ChatView", "MessageReceived") => "chime.wav",
            _ => null
        };

        if (sound is not null)
            PlaySound(sound);
    }
}
```

## Standard MAUI Control Integration

Beyond Shiny's own controls, you can opt in to automatic feedback for standard MAUI controls — Button, Entry, Slider, Switch, and more. This uses `Application.DescendantAdded`/`DescendantRemoved` to hook into the visual tree lifecycle, binding and unbinding events automatically with no modifications to the controls themselves.

```csharp
builder.UseShinyControls(cfg =>
{
    cfg.AddDefaultMauiControlFeedback();
});
```

### Integrated MAUI Controls

| Control | Event | Details | Description |
|---|---|---|---|
| `Button` | `Clicked` | — | Button tapped |
| `Entry` | `TextChanged` | new text value | Text input changed |
| `Slider` | `ValueChanged` | new value | Slider moved |
| `Switch` | `Toggled` | value | Switch toggled |
| `CheckBox` | `CheckedChanged` | value | Checkbox toggled |
| `DatePicker` | `DateSelected` | new date | Date selected |
| `TimePicker` | `TimeChanged` | — | Time changed |
| `Picker` | `SelectedIndexChanged` | — | Selection changed |
| `SearchBar` | `SearchButtonPressed` | — | Search submitted |
| `Stepper` | `ValueChanged` | new value | Stepper changed |
| `Editor` | `TextChanged` | new text value | Editor text changed |
| `RadioButton` | `CheckedChanged` | value | Radio button toggled |

Events are properly unhooked when controls are removed from the visual tree, ensuring no memory leaks.

## Disabling Feedback

Set `UseFeedback="False"` on any control to suppress feedback for that instance:

```xml
<shiny:FloatingPanel UseFeedback="False" ... />
<shiny:SecurityPin UseFeedback="False" ... />
```

For toast:
```csharp
await toaster.ShowAsync("Silent toast", cfg => cfg.UseFeedback = false);
```
