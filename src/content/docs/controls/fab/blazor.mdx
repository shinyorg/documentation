---
title: Fab & FabMenu | Blazor Usage
description: Using Fab and FabMenu in Blazor applications
---

The Blazor `<Fab>` and `<FabMenu>` components mirror the MAUI controls with the same properties and behavior.

## Setup

```razor
@using Shiny.Blazor.Controls
```

## Fab

```razor
<div style="position: relative; height: 100vh;">
    <!-- Page content -->

    <!-- Icon-only Fab -->
    <Fab IconUrl="add.png"
         FabBackgroundColor="#E91E63"
         Command="OnAdd" />

    <!-- Extended Fab with text -->
    <Fab IconUrl="add.png"
         Text="Add Item"
         FabBackgroundColor="#4CAF50"
         TextColor="white"
         Command="OnAdd" />
</div>

@code {
    Task OnAdd()
    {
        // Handle tap
        return Task.CompletedTask;
    }
}
```

## FabMenu

```razor
<div style="position: relative; height: 100vh;">
    <!-- Page content -->

    <FabMenu IsOpen="@isMenuOpen"
             IsOpenChanged="v => isMenuOpen = v"
             IconUrl="plus.png"
             FabBackgroundColor="#2196F3"
             HasBackdrop="true">
        <FabMenuItem IconUrl="share.png"
                     Text="Share"
                     FabBackgroundColor="#4CAF50"
                     Command="OnShare" />
        <FabMenuItem IconUrl="edit.png"
                     Text="Edit"
                     FabBackgroundColor="#FF9800"
                     Command="OnEdit" />
        <FabMenuItem IconUrl="delete.png"
                     Text="Delete"
                     FabBackgroundColor="#F44336"
                     Command="OnDelete" />
    </FabMenu>
</div>

@code {
    bool isMenuOpen;

    Task OnShare() => Task.CompletedTask;
    Task OnEdit() => Task.CompletedTask;
    Task OnDelete() => Task.CompletedTask;
}
```

:::note
On Blazor, `Icon` becomes `IconUrl` (a URL string instead of `ImageSource`). Colors are CSS strings. `Command` is `EventCallback` instead of `ICommand`. `IsOpen` uses `IsOpenChanged` callback for two-way binding.
:::
