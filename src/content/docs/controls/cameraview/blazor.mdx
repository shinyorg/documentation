`Shiny.Blazor.Controls.Camera` brings the CameraView to Blazor WebAssembly using the browser's `getUserMedia` / `MediaRecorder` / `BarcodeDetector` APIs. All frame analysis runs **in JavaScript** ‚Äî only flat `Detection` DTOs cross the interop boundary ‚Äî so the camera stays fast and trim-safe.

- <NugetDownloads name="Shiny.Blazor.Controls.Camera" />

## Installation

```bash
dotnet add package Shiny.Blazor.Controls.Camera
```

```razor
@using Shiny.Blazor.Controls.Camera
@using Shiny.Controls.Camera
```

> `getUserMedia` requires a **secure context** ‚Äî HTTPS, or `localhost` during development.

## Usage

```razor
<div style="position:relative;width:100%;max-width:480px;aspect-ratio:3/4;">
    <CameraView @ref="camera"
                Facing="CameraFacing.Back"
                EnableBarcode="true"
                ShowOverlay="true"
                Filter="filter"
                DetectionsChanged="OnDetections"
                OnError="msg => status = msg"
                Style="width:100%;height:100%;" />
</div>

<button @onclick="Capture">Photo</button>
<button @onclick="ToggleRecording">@(recording ? "Stop" : "Record")</button>

<select @bind="filter">
    @foreach (var f in Enum.GetValues<CameraFilter>())
    {
        <option value="@f">@f</option>
    }
</select>

@code {
    CameraView? camera;
    CameraFilter filter = CameraFilter.None;
    bool recording;
    string? status;

    void OnDetections(IReadOnlyList<Detection> dets)
    {
        var code = dets.FirstOrDefault(d => d.Type == DetectionType.Barcode);
        if (code?.Value is { } value)
            status = $"Barcode: {value}";
    }

    async Task Capture()
    {
        var jpeg = await camera!.CapturePhotoAsync();          // byte[]
        // ... use the JPEG bytes ...
    }

    async Task ToggleRecording()
    {
        if (!recording) { await camera!.StartRecordingAsync(); recording = true; }
        else { var webm = await camera!.StopRecordingAsync(); recording = false; }  // byte[] (WebM)
    }
}
```

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `Facing` | `CameraFacing` | `Back` | `Front` maps to the browser `"user"` facing mode |
| `EnableBarcode` | `bool` | `true` | Run the in-browser barcode detector |
| `ShowOverlay` | `bool` | `true` | Draw detection boxes on the overlay canvas |
| `AutoStart` | `bool` | `true` | Start the preview on first render |
| `Filter` | `CameraFilter` | `None` | Live CSS filter on the `<video>` element |
| `CssClass` / `Style` | `string?` | `null` | Host element styling |
| `DetectionsChanged` | `EventCallback<IReadOnlyList<Detection>>` | ‚Äî | Latest detections |
| `OnError` | `EventCallback<string>` | ‚Äî | Camera could not start (permission, no device, insecure context) |

### Methods

`StartAsync()` ¬∑ `StopAsync()` ¬∑ `CapturePhotoAsync()` ‚Üí JPEG `byte[]` ¬∑ `StartRecordingAsync(includeAudio = true)` ¬∑ `StopRecordingAsync()` ‚Üí WebM `byte[]`.

## Browser support

- **Barcode scanning** uses the native [`BarcodeDetector`](https://developer.mozilla.org/en-US/docs/Web/API/Barcode_Detection_API) ‚Äî available in Chromium browsers (Chrome / Edge). On unsupported browsers (Firefox / Safari) the component raises `OnError` once and the preview keeps running; slot in a ZXing-js fallback if you need universal coverage.
- **Filters** map to CSS `filter` strings, so they work in every browser (temperature filters are approximate ‚Äî CSS has no true white balance).
- **Recording** uses `MediaRecorder`; output is typically WebM.

For barcode generation/rendering (as opposed to scanning), see [Barcodes & QR Codes](/controls/barcodes/).
