The CameraView streams frames to a pipeline of **analyzers**. Each analyzer runs off the UI thread with per-analyzer back-pressure (a slow analyzer is simply skipped for a frame rather than backing up the camera). It surfaces its result through its **own strongly-typed event** (or a bindable `Command`) and returns the styled bounding boxes to draw as `OverlayBox`es in normalized, upright image space. The built-in `CameraOverlayView` draws them.

Two channels per analyzer:

- **Semantic result** ŌåÆ a typed `event` (e.g. `BarcodeDetected`) or a bindable `Command`, raised on the UI thread.
- **Presentation** ŌåÆ the boxes returned from `AnalyzeAsync`. A returned set persists until the analyzer returns a different set (replace) or `null` (clear).

## The pipeline

```csharp
var barcode = new BarcodeAnalyzer();
barcode.BarcodeDetected += (_, e) => status = $"{e.Format}: {e.Value}";

var motion = new MotionAnalyzer();
motion.MotionChanged += (_, e) => status = e.InMotion ? "Motion" : "Still";

camera.Analyzers.Add(barcode);
camera.Analyzers.Add(motion);
```

Add or remove analyzers at any time ŌĆö the running session picks up the change. Analyzer events are raised on the UI thread (the pipeline marshals them), so handlers can touch UI directly.

## Declaring analyzers in XAML (+ Commands)

Analyzers are `BindableObject`s, so you can declare them inside `<cam:CameraView>` (its content property is `Analyzers`) under the one `cam:` prefix, and bind each analyzer's **`ŌĆ”Command`** to a ViewModel. They inherit the camera's `BindingContext`, and each command fires with the same args as its event.

```xml
<cam:CameraView Facing="Back" Filter="Chrome">
    <cam:BarcodeAnalyzer BarcodeDetectedCommand="{Binding ScanCommand}" />
    <cam:InvoiceAnalyzer DocumentDetectedCommand="{Binding InvoiceCommand}" />
    <!-- run an analyzer for its command only, no box: -->
    <cam:MotionAnalyzer MotionChangedCommand="{Binding MotionCommand}" ShowBoundingBox="False" />
</cam:CameraView>
```

```csharp
public ICommand ScanCommand { get; } =
    new Command<BarcodeDetectedEventArgs>(e => /* e.Format, e.Value, e.BoundingBox */);
```

## Drawing the overlay

Drop a `CameraOverlayView` over the `CameraView` in the same cell and point it at the camera ŌĆö it auto-subscribes to the aggregated boxes and redraws. Each analyzer styles its own boxes (color / text); `DefaultBoxColor` / `DefaultTextColor` fill in anything unset.

```xml
<Grid>
    <cam:CameraView x:Name="Camera" ScaleMode="AspectFill" />
    <cam:CameraOverlayView Camera="{x:Reference Camera}" InputTransparent="True" />
</Grid>
```

### Enabling / disabling analyzers

`Analyzers` is an observable collection ŌĆö **add or remove at runtime and the running pipeline picks it up live** (seamless on Apple/Windows; Android rebinds its capture use-cases automatically). To turn an analyzer off *without losing its bindings or internal state*, set **`FrameAnalyzer.IsEnabled`** (bool, default `true`) instead of removing it ŌĆö it resumes instantly when re-enabled. When **every** analyzer is disabled the camera behaves as if it had none (so, e.g., Android can record video again).

```xml
<!-- toggle live from a switch; the analyzer's command binding + state are preserved while off -->
<cam:BarcodeAnalyzer BarcodeDetectedCommand="{Binding ScanCommand}"
                     IsEnabled="{Binding IsToggled, Source={x:Reference ScanSwitch}}" />
```

### Controlling the overlay per analyzer

- **`IsEnabled`** (bool, default `true`) ŌĆö run the analyzer or not (see above).
- **`ShowBoundingBox`** (bool, default `true`) ŌĆö set `False` to run an analyzer purely for its event/command and draw nothing. Honored by every built-in analyzer.
- **`OverlayProvider`** (code-level `Func<TArgs, IReadOnlyList<OverlayBox>?>`) ŌĆö return the exact boxes to draw for a detection, or `null` for none. When unset, the analyzer draws its own default styled box.

```csharp
barcode.OverlayProvider = e => e.Value.StartsWith("OK")
    ? [ new OverlayBox(e.BoundingBox, Colors.Lime, e.Value) ]
    : null;   // don't box anything that doesn't start with "OK"
```

`CoordinateTransform` (in `Shiny.Controls.Camera`) maps normalized boxes into view space accounting for aspect-fill crop, rotation and front-camera mirroring ŌĆö the same helper the overlay uses.

## Capture & stop on detection

A common flow is "scan until you get a result, then freeze the preview and keep the photo." Every analyzer (it's on the `FrameAnalyzer` base) exposes two flags for this ŌĆö no manual wiring:

- **`CaptureOnDetection`** (bool, default `false`) ŌĆö capture a full-res still the moment the analyzer **confirms** a detection.
- **`StopOnDetection`** (bool, default `false`) ŌĆö stop the capture session on a confirmed detection.

The result is delivered on **`CameraView.DetectionCaptured`** (`DetectionCapturedEventArgs`: the `Analyzer`, the `Detection` payload, and the `Photo`):

```xml
<cam:CameraView x:Name="Camera" DetectionCaptured="OnDetectionCaptured">
    <cam:PassportAnalyzer CaptureOnDetection="True" StopOnDetection="True" />
</cam:CameraView>
```

```csharp
void OnDetectionCaptured(object? sender, DetectionCapturedEventArgs e)
{
    var photo = e.Photo;                        // full-res still (null if only StopOnDetection)
    if (e.Detection is Passport p) { /* use the merged document */ }
}
```

"Confirmed" means the same thing the analyzer's typed event uses ŌĆö for documents it's the **merged** record (after accumulation), for `MotionAnalyzer` it's motion **starting**. The trigger **latches** so a flurry of detections (or several analyzers firing at once) captures once; `StartAsync()` re-arms it. Capture-only (no stop) keeps firing on later detections.

Prefer to drive it yourself? Call **`await Camera.CaptureAndStopAsync()`** from any handler ŌĆö it grabs the full-res still and stops the session atomically, returning the `CameraPhoto`.

## Analyzer packages

Each analyzer ships as its own package so apps pull only what they need. The strategy is **hybrid**: native ML where available, managed fallback elsewhere.

### Barcode

- <NugetDownloads name="Shiny.Maui.Controls.Camera.Barcode" />

```csharp
using Shiny.Maui.Controls.Camera.Barcode;

var barcode = new BarcodeAnalyzer();
barcode.BarcodeDetected += (_, e) =>
    Console.WriteLine($"{e.Format}: {e.Value}");   // e.BoundingBox is normalized upright
camera.Analyzers.Add(barcode);
```

Decodes 1D/2D barcodes and QR codes with the **native scanner** ŌĆö Apple Vision (`VNDetectBarcodesRequest`) on iOS / Mac Catalyst / macOS and Android MLKit (`BarcodeScanning`). A no-op on Windows and bare `net10.0` (no native barcode scanner). `BarcodeDetectedEventArgs` carries the `Format` (`Shiny.Controls.Camera.BarcodeFormat`), `Value`, and `BoundingBox`, and fires **once per barcode** in view.

Restrict the scanned symbologies with `BarcodeAnalyzer.Formats` (`IList<BarcodeFormat>?`, null = all) ŌĆö the filter is applied natively (Vision `Symbologies` / MLKit `SetBarcodeFormats`). In XAML it's settable inline as a case-insensitive, comma-separated list:

```xml
<cam:BarcodeAnalyzer BarcodeDetectedCommand="{Binding ScanCommand}"
                     Formats="QrCode,Ean13,Code128" />
```

Set `Formats` before scanning starts (constructor / XAML); on Android the scanner client is built on the first frame, so changing it mid-session takes effect only after the analyzer is re-created.

### Motion

- <NugetDownloads name="Shiny.Maui.Controls.Camera.Motion" />

```csharp
using Shiny.Maui.Controls.Camera.Motion;

var motion = new MotionAnalyzer
{
    PixelThreshold = 25,   // per-pixel luma delta to count as changed
    AreaThreshold = 0.04,  // fraction of pixels that must change to trigger
    GridColumns = 16,      // grid resolution used to split motion into regions
    CellThreshold = 0.10,  // fraction of a cell that must change to box it
    EnterFrames = 3,       // consecutive frames above threshold before reporting "started"
    ExitFrames = 5         // consecutive frames below threshold before reporting "stopped"
};
// e.Regions = a box per distinct moving area; e.Region = their union; e.Intensity = 0..1
motion.MotionChanged += (_, e) => { /* e.InMotion, e.Regions, e.Region, e.Intensity */ };
camera.Analyzers.Add(motion);
```

Pure-managed luminance frame-differencing ŌĆö cross-platform, no native dependency. Raises `MotionChanged` when motion starts and stops, and **clusters motion into separate regions** so movement in two spots draws two boxes rather than one box spanning both. `MotionEventArgs.Regions` carries one normalized `RectF` per moving area (with `Region` as their union for back-compat); tune how finely motion is split with `GridColumns` / `CellThreshold`.

The event is **debounced** so it isn't twitchy: `MotionChanged` reports `InMotion=true` only after `EnterFrames` consecutive frames above threshold (default 3) and `InMotion=false` only after `ExitFrames` below it (default 5) ŌĆö a single noisy frame or a brief pause mid-movement won't fire it. Raise `EnterFrames` if it's still too sensitive. The overlay boxes are **not** debounced, so they keep tracking each frame.

### Documents ŌĆö invoices, receipts, IDs, cards, passports

- <NugetDownloads name="Shiny.Maui.Controls.Camera.Documents" />

Structured extraction, where **each document type is its own analyzer with its own strongly-typed event** (`DocumentDetected`, typed to the document). Every payload is a strong record with **nullable fields** ŌĆö only what was found is populated. Ships analyzers for **invoices** (`Invoice`, with order lines), **receipts** (`Receipt`, with line items + per-tax breakdown + totals), **driver's licenses** (`DriversLicense`), **health cards** (`HealthCard`), **credit cards** (`CreditCard`), and **passports** (`Passport`).

:::note[OCR runs once per frame]
The document analyzers and `OcrAnalyzer` share a single OCR pass per frame (the recognizer caches its result on the frame instance), so enabling several document analyzers together does **one** recognition per frame, not one each. The shared pass runs with Vision language correction **off** ŌĆö it otherwise corrupts structured fields (license/MRZ/card numbers, totals, dates) by snapping codes to dictionary words; the parsers fuzzy-match the raw text.
:::

:::tip[Documents are deskewed before OCR]
The document analyzers detect the document, **perspective-correct (deskew) it**, then OCR the flat crop ŌĆö flattening an angled card/ID dramatically improves accuracy, since keystoned text reads poorly. This happens **inside the live pipeline**, so your `CameraView` preview is untouched (it's not the modal document-scanner UI), and the overlay becomes the detected document outline (`DocumentAnalyzer.BoxColor`). Per platform:

- **iOS / Mac Catalyst / macOS** ŌĆö Vision (`VNDetectDocumentSegmentationRequest`) + Core Image `CIPerspectiveCorrection`
- **Windows** ŌĆö OpenCvSharp (Canny ŌåÆ largest convex 4-gon ŌåÆ `WarpPerspective`)
- **Android** ŌĆö a dependency-free managed detector (Otsu + largest bright region + extreme corners) plus a native `Matrix.setPolyToPoly` warp ŌĆö no OpenCV, so the package stays trim/AOT-clean
- **bare net10.0** ŌĆö no-op

When no document is found, all platforms fall back to whole-frame OCR.
:::

:::tip[Fields accumulate across frames]
As a document slides into focus its fields appear gradually ŌĆö number first, then name, then expiry. Rather than firing `DocumentDetected` on every frame with a stream of partial objects, the analyzer **merges the reads over up to `AccumulationFrames` frames** (default 5) and fires **once** with the richest combined record. It fires early when the parse is complete, won't re-fire while the same document stays in view, and re-arms after `ResetAfterEmptyFrames` frames with nothing detected. Set `AccumulationFrames = 1` for the old fire-every-frame behavior. (`DriversLicenseAnalyzer` reads a PDF417 barcode ŌĆö complete in one shot ŌĆö so it isn't accumulated.)
:::

```csharp
using Shiny.Maui.Controls.Camera.Documents;

var invoice = new InvoiceAnalyzer();
invoice.DocumentDetected += (_, e) =>
{
    Invoice doc = e.Document;
    Console.WriteLine($"#{doc.Number} total {doc.Total} ŌĆö {doc.Lines.Count} line(s)");
    foreach (var line in doc.Lines)        // InvoiceLine { Description, Quantity, UnitPrice, Amount }
        Console.WriteLine($"  {line.Quantity} x {line.Description} = {line.Amount}");
};

var license = new DriversLicenseAnalyzer();   // deterministic ŌĆö no ML
license.DocumentDetected += (_, e) =>
    Console.WriteLine($"{e.Document.FirstName} {e.Document.LastName} ŌĆö {e.Document.Number}");

camera.Analyzers.Add(invoice);
camera.Analyzers.Add(license);
```

- **`DriversLicenseAnalyzer`** reads the **PDF417** barcode on the back of US/Canadian licenses with the native scanner (Apple Vision / Android MLKit; a no-op on Windows and bare `net10.0`) and parses the **AAMVA** record into a strongly-typed `DriversLicense` (number, names, DOB, expiry, address, `Jurisdiction`) ŌĆö the AAMVA parse is deterministic. It covers US states and the Canadian provinces that emit an AAMVA PDF417 (BC, AB, SK, MB, NS, NB, PEI, NL) and auto-detects Canadian `CCYYMMDD` date order from the country element or the province code (so dates parse correctly even when the country field is absent). **Ontario and Quebec licences do not carry a PDF417 barcode and therefore cannot be scanned this way** ŌĆö back an OCR-based `DocumentAnalyzer` with a custom parser for those.
- **`PassportAnalyzer`** locates and parses the passport **MRZ** (the two `<<<` lines, ICAO TD3) into a `Passport` (number, surname, given names, nationality, issuing country, DOB, expiry, sex) ŌĆö the MRZ parse is deterministic, only locating the lines depends on OCR.
- **`CreditCardAnalyzer`** reads the front of a payment card into a `CreditCard`. The **brand** (`CreditCardType` ŌĆö Visa, Mastercard, Amex, ŌĆ”) and number validity are derived deterministically from the number's IIN prefix + Luhn; name / expiry / company are best-effort OCR. (`Cvv` is on the back signature panel and PCI-sensitive ŌĆö it is almost always `null` from a front scan.)
- **`ReceiptAnalyzer`** reads a point-of-sale receipt into a `Receipt` ŌĆö merchant header, purchased line items (`Receipt.Lines`), a **per-tax breakdown** (`Receipt.Taxes`, each with an optional rate), plus `Subtotal` / `Tax` (the sum) / `Tip` / `Discount` / `Total` and best-effort `PaymentMethod` / `CardLast4` / `Currency` / `Date` / `Time`. OCR + best-effort rules.
- **`HealthCardAnalyzer`** is OCR + best-effort rules tuned for **Canadian** health cards: it detects the issuing province from on-card keywords and applies that province's member-number format ŌĆö Quebec/RAMQ (4 letters + 8 digits), Ontario/OHIP (10 digits + a 2-letter version code), BC Personal Health Number (10 digits), Alberta/AHCIP (9 digits), and the rest of the provinces ŌĆö surfacing `HealthCard.Province` plus a `Plan` field. Unknown layouts fall back to the longest plausible digit run.
- **`InvoiceAnalyzer`** / **`ReceiptAnalyzer`** / **`HealthCardAnalyzer`** are OCR + best-effort rules. Swap the rules on any OCR-backed analyzer by supplying a custom `IDocumentParser<T>` to the constructor (`new InvoiceAnalyzer(new MyInvoiceParser())`), where `TryParse` returns the typed payload plus the boxes to draw ŌĆö back it with rules, a cloud Document AI, or a vision LLM.

```csharp
var card = new CreditCardAnalyzer();
card.DocumentDetected += (_, e) =>
    Console.WriteLine($"{e.Document.Type} {e.Document.Number} exp {e.Document.Expiry:MM/yy}");

var passport = new PassportAnalyzer();    // deterministic MRZ parse
passport.DocumentDetected += (_, e) =>
    Console.WriteLine($"{e.Document.GivenNames} {e.Document.Surname} ({e.Document.Nationality})");
```

The reusable building blocks (`RecognizedText`, `DocumentField`, `DocumentLineItem`, `DocumentDetectedEventArgs<T>`, `IDocumentParser<T>`) live in `Shiny.Controls.Camera`.

#### Document payloads

Every document analyzer hands back a **strongly-typed record**. Every data property is **nullable** (or a `Unknown`/`Unspecified` enum) ŌĆö only what was found is populated, so check before use.

record InvoiceLine(string? Description, decimal? Quantity, decimal? UnitPrice, decimal? Amount, RectF? Bounds);

record Receipt(string? Merchant, string? MerchantPhone, string? ReceiptNumber,
               DateOnly? Date, TimeOnly? Time, IReadOnlyList<ReceiptLine> Lines,
               decimal? Subtotal, IReadOnlyList<ReceiptTax> Taxes, decimal? Tax,
               decimal? Tip, decimal? Discount, decimal? Total,
               string? Currency, string? PaymentMethod, string? CardLast4,
               IReadOnlyList<DocumentField> Fields);
record ReceiptLine(string? Description, decimal? Quantity, decimal? UnitPrice, decimal? Amount, RectF? Bounds);
record ReceiptTax(string? Label, decimal? Rate, decimal? Amount, RectF? Bounds);

record DriversLicense(string? Number, string? FirstName, string? LastName,
                      DateOnly? DateOfBirth, DateOnly? Expiry, string? Address,
                      string? Jurisdiction, IReadOnlyList<DocumentField> Fields);

record HealthCard(string? Number, string? Name, DateOnly? Expiry, string? Issuer,
                  string? Province, IReadOnlyList<DocumentField> Fields);

record CreditCard(CreditCardType Type, string? Number, DateOnly? Expiry,
                  string? FirstName, string? LastName, string? CompanyName, string? Cvv,
                  IReadOnlyList<DocumentField> Fields);
enum CreditCardType { Unknown, Visa, Mastercard, Amex, Discover, DinersClub, JCB, UnionPay, Maestro }

record Passport(string? Number, string? Surname, string? GivenNames,
                string? Nationality, string? IssuingCountry,
                DateOnly? DateOfBirth, DateOnly? Expiry, PassportSex Sex,

Beyond the first-class properties, every payload also exposes a `Fields` list of `DocumentField` (label/value/box/confidence) carrying anything extra the parser recognized ŌĆö including, for a custom `IDocumentParser<T>`, fields you don't model as typed properties.

## Writing your own analyzer

Derive from `FrameAnalyzer` (it implements `IFrameAnalyzer`, makes the analyzer a `BindableObject`, marshals events/commands to the UI thread, and adds `IsEnabled` + `ShowBoundingBox` + `OverlayProvider`). Use `frame.GetLuminance()` for a cached grayscale plane (ideal for managed CV), or down-cast `CameraFrame` to the platform frame (`AppleCameraFrame`, `AndroidCameraFrame`, `WindowsCameraFrame`) to feed a native ML SDK with zero copies.

```csharp
public class GreenScreenAnalyzer : FrameAnalyzer
{
    public override string Id => "myapp.greenscreen";

    public event EventHandler<GreenScreenEventArgs>? Detected;

    public override ValueTask<IReadOnlyList<OverlayBox>?> AnalyzeAsync(CameraFrame frame, CancellationToken ct)
    {
        var luma = frame.GetLuminance();   // ReadOnlySpan<byte>, Width x Height
        // ... your detection logic ...
        if (/* nothing found */ true)
            return default;                 // null -> clear this analyzer's boxes

        var args = new GreenScreenEventArgs(/* ... */);
        this.Emit(() => this.Detected?.Invoke(this, args), /* command */ null, args);

        return new ValueTask<IReadOnlyList<OverlayBox>?>(
            this.ResolveOverlay(args, /* OverlayProvider */ null,
                () => new[] { new OverlayBox(box) }));
    }
}
```

`Emit(raiseEvent, command, args)` raises your event and invokes a bound `Command` (both on the UI thread); `ResolveOverlay(args, provider, defaultBoxes)` honors `ShowBoundingBox` and any `OverlayProvider`. Keep analyzers allocation-light and don't retain the frame past the returned task ŌĆö the pipeline disposes it (releasing the pooled native buffer) once every analyzer finishes.

## Writing a custom document analyzer

For an **OCR-backed document** you don't derive from `FrameAnalyzer` directly ŌĆö derive from **`DocumentAnalyzer<TDocument>`** and supply an **`IDocumentParser<TDocument>`**. The base class does the heavy lifting: it runs the shared OCR recognizer each frame, hands the recognized text to your parser, raises the typed `DocumentDetected` event (and the bound `DocumentDetectedCommand`) on the UI thread, draws the parser's boxes, and honors `IsEnabled` / `ShowBoundingBox` / `OverlayProvider`. You only write **what counts as your document and how to pull its fields out of text**.

A parser is the single seam ŌĆö it turns the OCR'd lines into a typed payload plus the boxes to draw:

```csharp
public interface IDocumentParser<TDocument>
{
    // true + populated outputs on success; false when the text isn't this document (analyzer clears its overlay)
    bool TryParse(IReadOnlyList<RecognizedText> text,
                  out TDocument document,
                  out IReadOnlyList<OverlayBox> boxes);
}
```

`RecognizedText` (`Text`, `BoundingBox`, `Confidence`) is already in normalized **upright** image space, so boxes you build from `.BoundingBox` line up with the overlay automatically.

### Option A ŌĆö swap the parser on an existing analyzer

The quickest customization: keep the built-in analyzer (and its payload type) but replace the rules ŌĆö pass your own `IDocumentParser<T>` to its constructor. Back it with stricter rules, a cloud Document AI, or a vision LLM without touching the rest of the pipeline.

```csharp
camera.Analyzers.Add(new InvoiceAnalyzer(new MyInvoiceParser()));   // IDocumentParser<Invoice>
```

### Option B ŌĆö a brand-new document type

Three pieces: a **payload record**, a **parser**, and a one-line **analyzer** that wires them. Here's a business-card scanner end to end:

```csharp
using System.Text.RegularExpressions;
using Microsoft.Maui.Graphics;
using Shiny.Controls.Camera;                  // RecognizedText, DocumentField, IDocumentParser<T>, OverlayBox
using Shiny.Maui.Controls.Camera.Documents;   // DocumentAnalyzer<T>

// 1) payload ŌĆö nullable fields (only what was found is set) + a Fields bag for extras
public record BusinessCard(
    string? Name, string? Company, string? Email, string? Phone,
    IReadOnlyList<DocumentField> Fields);

// 2) parser ŌĆö pull fields out of the OCR'd lines
public sealed partial class BusinessCardParser : IDocumentParser<BusinessCard>
{
    [GeneratedRegex(@"[\w.+-]+@[\w-]+\.[\w.-]+")] private static partial Regex Email();
    [GeneratedRegex(@"\+?\d[\d ().-]{7,}\d")]     private static partial Regex Phone();

    public bool TryParse(
        IReadOnlyList<RecognizedText> text,
        out BusinessCard document,
        out IReadOnlyList<OverlayBox> boxes)
    {
        document = null!;
        boxes = [];

        // a cheap "is this my document?" signal first ŌĆö an email line. Bail fast otherwise so the
        // analyzer doesn't misfire on every frame of unrelated text.
        var emailLine = text.FirstOrDefault(t => Email().IsMatch(t.Text));
        if (emailLine is null)
            return false;

        var phoneLine = text.FirstOrDefault(t => Phone().IsMatch(t.Text));
        var name = text.FirstOrDefault()?.Text;                       // first line is usually the name
        var company = text.Skip(1).FirstOrDefault(t =>               // next non-contact line Ōēł company
            !Email().IsMatch(t.Text) && !Phone().IsMatch(t.Text))?.Text;

        var fields = new List<DocumentField>
        {
            new("Name", name, text.FirstOrDefault()?.BoundingBox),
            new("Company", company),
            new("Email", Email().Match(emailLine.Text).Value, emailLine.BoundingBox),
            new("Phone", phoneLine is null ? null : Phone().Match(phoneLine.Text).Value, phoneLine?.BoundingBox),
        };

        document = new BusinessCard(fields[0].Value, company, fields[2].Value, fields[3].Value, fields);

        // box each field we actually located
        boxes = fields
            .Where(f => f.Bounds is not null)
            .Select(f => new OverlayBox(f.Bounds!.Value, Colors.Lime, f.Label))
            .ToList();
        return true;
    }
}

// 3) analyzer ŌĆö the base does OCR + threading + events + overlay
public sealed class BusinessCardAnalyzer : DocumentAnalyzer<BusinessCard>
{
    public BusinessCardAnalyzer() : base(new BusinessCardParser()) { }
    public BusinessCardAnalyzer(IDocumentParser<BusinessCard> parser) : base(parser) { }  // allow swap-in too

    public override string Id => "myapp.camera.businesscard";
}
```

Use it like any other analyzer ŌĆö code or XAML (it inherits `DocumentDetectedCommand`, `IsEnabled`, `ShowBoundingBox`, and `OverlayProvider` from the base):

```csharp
var cards = new BusinessCardAnalyzer();
cards.DocumentDetected += (_, e) => status = $"{e.Document.Name} ŌĆö {e.Document.Email}";
camera.Analyzers.Add(cards);
```

```xml
<my:BusinessCardAnalyzer DocumentDetectedCommand="{Binding CardCommand}"
                         IsEnabled="{Binding ScanCards}" />
```

:::note[Things to keep in mind]
- **`TryParse` runs on the analysis thread**, once per frame the recognizer produces text. Keep it fast and allocation-light, and do no UI work ŌĆö the base marshals `DocumentDetected` to the UI thread for you.
- **Return `false` for "not my document."** Lead with a cheap signal check (a keyword, an email, an MRZ line) so you don't emit a half-parsed result on every frame of unrelated text.
- **OCR is native** (Apple Vision / Android MLKit / `Windows.Media.Ocr`), so a `DocumentAnalyzer<T>` produces results on iOS / Android / Windows / macOS ŌĆö not on bare `net10.0`.
- **A remote/LLM parser is fine** ŌĆö the pipeline runs one analysis at a time per analyzer and drops frames while busy, so a slow `TryParse` just lowers that analyzer's throughput rather than backing up the camera. Throttle or cache if the call is expensive.
- **Consumers can still override the boxes** without touching your parser via the analyzer's `OverlayProvider` (`Func<TDocument, IReadOnlyList<OverlayBox>?>`).
- **Opt into frame accumulation** by overriding the two defaulted `IDocumentParser<T>` methods so the analyzer merges your document across frames before firing: `Merge(accumulated, incoming)` fills in fields seen on later frames (typically `accumulated.X ?? incoming.X`, returning `incoming` when it's a *different* document), and `IsComplete(document)` lets it fire early once the key fields are present. The defaults keep the old fire-every-frame behavior, so existing parsers need no changes.
